# Phase 2: Simulation Engine & Real-Time Dashboard (Copilot Prompt Spec)

## 1. Project Overview & Objective
This document outlines the architecture for **Phase 2: Simulate** of the HP Metal Jet S100 Digital Twin. 
**Goal:** Generate a decoupled, real-time Python simulation architecture using `asyncio` for the logic engine and `Streamlit` for the UI. The simulation must advance time, generate realistic synthetic telemetry, call the Phase 1 mathematical model, and persist all state changes to a database.

## 2. Architectural Blueprint (Decoupled System)
The system MUST be split into two separate Python scripts to prevent the UI from freezing during simulation loops.

### 2.1 The Database (The Historian)
* Use `sqlite3` or `SQLAlchemy` with SQLite.
* **Table:** `telemetry_log`
* **Columns:** `id`, `timestamp` (datetime), `run_id` (string), `temperature` (float), `load` (float), `contamination` (float), `blade_health` (float), `nozzle_health` (float), `heater_health` (float), `failure_log` (string/nullable).

### 2.2 Script A: `engine.py` (The Clock & Data Generator)
This script runs in the background. It must implement the `SimulationConfig` object containing:
* `Total Duration`: Total time to model.
* `Time Step`: Interval between updates (e.g., 1 simulated second/minute).
* `Environmental Profile`: The sequence of input drivers.

**Data Generation Logic:**
1. **State Machine:** Implement a class that tracks the printer's state:
   * `IDLE`: Temp = 25°C, Load = 0.
   * `HEATING`: Temp rises sharply, Load spikes.
   * `PRINTING`: Temp plateaus at target (e.g., 180°C), Load steady.
   * `COOLDOWN`: Temp drops exponentially.
2. **Gaussian Noise:** Apply `numpy.random.normal()` to the base state values at every tick to simulate real sensor variance.
3. **Execution Loop:** Use an `asyncio` loop (`while True`) that:
   * Calculates new input drivers based on State + Noise.
   * Calls the Phase 1 Engine to get updated component Health Indexes.
   * Inserts the full row (Inputs + Outputs + Timestamp) into the SQLite database.
   * Sleeps for a short real-world duration (e.g., `asyncio.sleep(1)`).

### 2.3 Script B: `app.py` (Streamlit Dashboard)
This script acts purely as a read-only viewer for the SQLite database. 

**Layout Requirements:**
1. **Sidebar (Simulation Controls):**
   * Sliders for "Base Temperature" and "Production Volume". (Write these preferences to a config file/DB that `engine.py` reads).
   * **Chaos Button:** `st.button("Inject Thermal Anomaly")`. When clicked, flag `engine.py` to spike the temperature.
2. **Top Row (System Health Metrics):**
   * Use `st.columns(3)`.
   * Display `st.metric()` for Recoater Blade, Nozzle Plate, and Heating Elements.
   * Calculate the `delta` by comparing the latest DB row with the previous DB row.
3. **Middle Row (Time Series Visualization):**
   * Fetch the last 100 rows from SQLite into a Pandas DataFrame.
   * **Chart 1:** `st.line_chart()` plotting environmental drivers (Temperature, Contamination).
   * **Chart 2:** `st.line_chart()` plotting the Health Index of the three components trending downwards.
4. **Bottom Row (Failure Analysis Log):**
   * Use `st.dataframe()` or `st.markdown()` to display events where a component health reached `0.0`.
   * Example string: `"[Timestamp] Nozzle Plate FAILED. Root cause: Thermal limits exceeded."`.
5. **The Heartbeat:**
   * At the end of the script, implement auto-refresh:
     ```python
     import time
     time.sleep(1)
     st.rerun()
     ```
